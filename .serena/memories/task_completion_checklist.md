# タスク完了時のチェックリスト

## コード生成・修正後の必須チェック

### 1. JSDocコメントの追加
- [ ] 関数やコンポーネントに**必ず**JSDocコメントを追加したか
- [ ] 必要最小限のコメントにとどめたか

### 2. 品質チェックの実行

```bash
# Lintチェック + 型チェック
pnpm lint && pnpm typecheck
```

CLAUDE.mdには`pnpm run ai-check`と記載されていますが、このスクリプトはpackage.jsonに未定義のため、上記の2つのコマンドを個別に実行します。

### 3. テストの実行

```bash
# 単体・結合テストを実行
pnpm test
```

- [ ] 既存のテストがすべてパスするか
- [ ] 新しいコードに対応するテストを追加したか

### 4. 実装前チェックリスト (契約による設計)

- [ ] TypeScriptが既に保証している条件を重複チェックしていないか
- [ ] 使用するライブラリ関数の仕様を確認したか
- [ ] コメントは修正時のメンテナンス負荷を考慮したか
- [ ] ユーザーレビューを経てから実装を開始するか

## コミット前の確認

- [ ] Lint/Format/型チェックがすべてパスしているか
- [ ] テストがすべてパスしているか
- [ ] 不要なコンソールログやデバッグコードを削除したか
- [ ] コミットメッセージは適切か

## MCP使用ガイドライン (CLAUDE.mdより)

1. **Serena MCP**: ファイル編集とロードに使用
2. **context7 MCP**: ライブラリの型定義やAPI仕様が不明な場合に使用
3. **DeepWiki MCP**: ライブラリで例外が発生した場合に使用
